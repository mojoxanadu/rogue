dragon-rs Feasibility Demo

This page uses the original live dragon-rs site as the render target to test whether its runtime is acceptable for our loading screen direction.

What This Tests
  • Visual quality of the original dragon-rs app
  • Whether it loads reliably in an embedded browser context
  • Whether WebGPU-heavy content is suitable for our offline/Edge target
  • Whether we should pursue a vendored integration later
Important

This is a feasibility harness, not a production integration. It depends on the live remote site and internet access. The main game still does not embed dragon-rs.

Original runtime source: https://hucanco.de/dragon-rs

Observed Constraints
  • Original app appears to be Svelte/WebGPU-based
  • Likely not suitable for direct single-file offline vendoring without significant work
  • Best next step, if approved, is separate asset/runtime experiment rather than folding into main build immediately
